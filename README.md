# What we can do?⠀⠀⠀⠀⠀⠀
### 1. DOM XSS Sink Detection
Identifies high-risk JavaScript sinks and execution contexts that often lead to DOM-based XSS.
* **Signatures:** `innerHTML`, `document.write()`, `eval()`, `dangerouslySetInnerHTML`, `location.href`.
* **Usage:** Pinpoints exact lines where user input might be handled unsafely, allowing for targeted exploitation.

### 2. Hardcoded Secret Scanning
Scans client-side code for accidental leakage of sensitive credentials and tokens.
* **Signatures:** AWS Access Keys (`AKIA...`), JWTs, generic API keys, and auth tokens.
* **Usage:** critical for identifying immediate account takeover risks or cloud infrastructure compromise.

### 3. Hidden Endpoint Extraction
Discovers unlinked API routes and internal paths not visible in the main UI navigation.
* **Signatures:** REST endpoints (e.g., `/api/v1/admin`), staging URLs, and internal service paths.
* **Usage:** Expands the attack surface by revealing shadow APIs or administrative routes hidden in the frontend bundles.

### 4. Parameter Mining (Logic & IDOR)
Extracts variable names and parameters used in API calls, essential for testing Business Logic flaws.
* **Signatures:** `user_id=`, `admin=`, `role:`, `permissions:`, `debug=`.
* **Usage:** Provides context for Mass Assignment or IDOR attacks (e.g., changing `role: "user"` to `role: "admin"` during interception).

### Engine Capabilities
* **Minification Handling:** automatically detects and optimizes analysis for minified files to prevent browser freezing.
* **Dual Input Mode:** Supports direct raw text input or `.js` file uploads.
* **JSON Export:** Generates structured logs for reporting or integration with other tools.
